CONTEXT

For food enthusiasts, sometimes we like to go back to excellent addresses that we have tried before : you don't change a winning team ! We don't want a new restaurant address, we want one we already know and loved! But it was so long ago...we don't remember the name of the restaurant...and this one, we already went there, but we don't know if the food was good, how can we remember? For example, when someone comes to visit us and we would like to take him/her to that great restaurant we went to a long time ago...what's its name again? Or our friend would prefer Japanese food, where can I take him to eat something of sure value? No question of taking risks. Sometimes we don't want to rely on the opinions of restaurant websites, we prefer to rely on our own experiences: we trust our tastes! How can we remember those good restaurants we went to in the past, and how can we not go back to those we didn't like if we don't remember?

SOLUTION

Why not create your own restaurant guide? We can only trust the address when it comes to our own opinions! There are so many cities where we have been, and so many restaurants that we don't remember the name or address when we went to Italy, or when we went to Paris, or even restaurants in our own city where we forget to go when we look for somewhere to eat. With a database system, it would be possible to keep personal notes, on our opinion about a restaurant where we have just eaten.
We could then easily access this information on a website, from any device. It would be interesting to classify the restaurants among those we liked and hated, or by specialty: an Italian, Chinese, Japanese, French, Spanish, ... restaurant. All that will be left to do is to create a "search" section in the site, where you can find certain restaurants in the database according to their location or the type of cuisine for example.

HOW TO CODE

In the following lines we will explain the required steps to code this project:

  • First you need to create a database where to store and retrieve your data.
  • Then you need to create a form to enter the data into the database.
  • Finally you need to save the data from the form into the database.


First create a database
What is a database ?
A database, allows the storage and retrieval of raw data or information, often related to a theme or activity; these can be of different natures and more or less related to each other.
If you want to learn more about the subject, you can click on this link or watch this explanatory video :

For this project we decided to create a Firebase database, as a free plan is available and it has good API to interact with the data. To create a Firebase database, you first need to create an account on Firebase, in order to use it later to save and retrieve data from it.



Now we create a form
For this project we created a form which you can access by clicking on the following link Demo (also in the navigation bar). This form allows you to enter the data of a new restaurant. The layout construction has been made simple by using the Skeleton framework, which in particular allows to obtain responsive pages.
Within this form each input field has its own id, which will allow us to esily get the data using jquery.



Finally save the data
